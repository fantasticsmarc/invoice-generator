# Contributing

Thanks for your interest in contributing to this project.

## Getting started
1. Fork the repository
2. Create a new branch for your change
3. Implement and test your changes locally
4. Open a Pull Request with a clear description

## Guidelines
- Keep the code readable and easy to maintain
- Prefer small, focused commits
- Update the documentation when behavior changes
- Avoid introducing breaking changes without documenting them

## Reporting issues
When opening an issue, include:
- A clear summary
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots or logs if relevant

## Suggestions for contributors
Useful contribution areas include:
- Input validation
- UI improvements
- Better invoice formatting
- Save/export enhancements
- Tax and subtotal logic improvements
